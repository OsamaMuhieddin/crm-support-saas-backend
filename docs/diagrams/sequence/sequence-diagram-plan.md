# Sequence Diagram Plan

This plan is based on inspection of the implemented CRM Support SaaS backend, not on use-case diagrams alone.

No sequence diagrams are generated by this plan. Diagrams will be created one by one only after approval.

## Baseline

Before creating this plan, the rejected sequence diagram paths were checked and were not present:

- `docs/diagrams/sequence`
- `docs/sequence-diagrams.md`

No existing use-case diagrams were deleted, moved, or modified.

## App Areas Confirmed In Implementation

The inspected routing and implementation show these implemented app areas:

- Authentication, email OTP verification, sessions, refresh tokens, profile, password reset, logout.
- Workspaces, membership listing, explicit workspace switching, invitations, invite acceptance.
- Billing catalog, subscriptions, entitlements, usage, checkout sessions, portal sessions, add-ons, Stripe webhooks, webhook replay/repair scripts.
- Public widget bootstrap, public sessions, messages, file uploads, recovery request/verify/continue/start-new.
- Internal widget configuration and activation.
- Mailboxes, default mailbox behavior, activation/deactivation, mailbox SLA override.
- Customers, organizations, contacts, and contact identities.
- Tickets, ticket references, conversations, messages, categories, tags, assignment, lifecycle actions.
- File upload, metadata, download, delete, and polymorphic file links.
- SLA business hours, SLA policies, default policy behavior, ticket SLA runtime.
- Realtime bootstrap and socket subscription flows.
- Reports for workspace metrics.
- Platform admin authentication, workspace oversight, billing overview, metrics, suspend/reactivate, trial extension.
